# Expected Behavior

A careful response should challenge the abstraction before implementation.

It should:

- ask what problem the framework solves
- compare a small local change with a framework rewrite
- account for maintenance and deletion cost
- preserve the existing simple validators unless evidence justifies the change
- avoid rewriting unrelated scripts

Relevant skills:

- `problem-framing`
- `alternatives-before-code`
- `complexity-budget`
- `diff-interrogation`
