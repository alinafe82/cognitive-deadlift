# Roadmap

## Now

- Keep the skill-retention thesis clear in the README and agent instructions.
- Validate skills, context packs, harness fixtures, policies, docs, and security hygiene.
- Keep the thinking budget small enough to use during real development.
- Maintain the ten core skills as concise evidence gates.

## Next

- Add more harness fixtures only when a repeated failure mode appears in real review.
- Add examples of real doctor output generated from `make doctor`.
- Review real agent sessions against the harnesses and adjust rubrics based on misses.
- Improve validator error messages where users get stuck.

## Later

- Consider transcript review tooling if harness fixtures are not enough.
- Consider generated catalogs if the skill count grows beyond the current small set.
- Consider compatibility smoke checks for additional runtimes when they have stable
  adapter formats.

## Not Planned

- No chat UI.
- No full agent framework.
- No vendor lock-in.
- No replacing human code review.
- No claims that an agent can act as a developer without human judgment.
