# Thinking Ledger

## Problem

The project has credible skills, validation, licensing, and plugin packaging, but no clear paid offer. A buyer cannot tell what to purchase, what it costs, what they get, or how to start.

## Current Evidence

- `README.md` explains the repo, skills, adapters, validation, and limitations.
- `CONTEXT.md` defines Cognitive Deadlift, Autopilot, Thinking Ledger, Friction, Skill, Hook, Plugin, Runtime Adapter, and Reasoning Evidence.
- `LICENSE-CONTENT` says commercial licensing requires separate permission through the maintainer's GitHub profile.
- `docs/adr/0000-one-public-plugin-repo.md` says the repo is the canonical public plugin package.
- `rg` found no existing pricing or paid offer page.

## Assumptions

- The most sellable first offer is a fixed-scope service, not a paid plugin SKU.
- A fixed price helps buyers decide faster than "contact for pricing."
- The offer should avoid claims that conflict with repo limitations, especially security audit or guaranteed AI behavior claims.
- The buying action can use the maintainer GitHub profile already present in plugin metadata.

## Alternatives

- Add a paid plugin tier. Rejected for now because packaging, distribution, support, and commercial licensing details are not resolved in the repo.
- Add only a commercial licensing page. Rejected because it would not explain the buyer problem, deliverables, or workflow outcome.
- Add an open-ended consulting page. Rejected because it is less sellable than a named fixed-scope audit.

## Chosen Approach

Add a `Cognitive Deadlift Workflow Audit` as a fixed-fee paid offer. Link it from the README, define `Workflow Audit` in the glossary, and keep the page focused on buyer fit, scope, deliverables, exclusions, price, and next action.

## Verification

Run `python3 scripts/validate_repo.py` to confirm the required repo structure and plugin manifests still validate.

## Residual Risk

USD 2,500 may be too low or too high for the intended buyer. The next revenue action should test the offer with one real prospect or one public issue/README call for feedback before expanding the pricing model.
